LispMark is a mostly complete, public domain™ language.

It can already handle many HTML tags, and will soon support user-definable macros.

The special characters ["(", ")", " "] are \ escapable. \\ escapes \ .

# Inspiration

The inspirations for this project are plenty:

  • Sexpcode, for trying to replace BBcode with SEXPs,
  • Arc, for trying to bring backend webdev into LISP,
  • Lis.py, for illustrating a basic LISP in python,
  • Curl, for trying to use list-based markup,
  • Bach, for trying to do these in Python

# Differences

Unlike Markdown, LispMark …

  • is easily extendible through macros
  • permits normal usage of normal characters
  • has non-ambiguous syntax
  • has few special characters
  • includes HTML anchors out of the box
  • etc.

Unlike BBcode, Lispmark's syntax is not mind-numbing, and is comparatively abstract.

Unlike Sexpcode, targeted at replacing BBcode, LispMark will intend to generally replace human writing of HTML.

# Supported HTML

Lispmark currently supports the following tags:

Tag name Meaning
(b text) Bold
(i text) Italics
(u text) Underline
(o text) Overline
(s text) Strikethrough
(sp text) Spoiler
(tt text) Teletype
(q text) Quotes text
(img src [title]) Creates an image with optional title.
(url href) Creates a transparent link.
(link href [text]) Creates hyperlink with optional title

# Composing functions

Composing a new function can be done like so: 

(def title (h2 (anc {0}) (link #view #) {1}))
By making use of the (def) tag, users then specify a LispMark template which can make use of the placeholder symbols {0} and {1}.

The first argument of the (def) command is the name of the new command, while the other contains the template for outputted text.

We can define a "thumbnail" function like this: 

(def thumb 
 (link {0} (img {0} {1}) (br) {1} (br) (br))
... where we use (thumb) to embed the image location given as the first argument, followed by the text given as the second argument, where both the image and text link to the original image.
